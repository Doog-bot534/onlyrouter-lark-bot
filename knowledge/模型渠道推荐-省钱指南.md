# 模型渠道推荐（省钱 + 跨协议避坑，已实测）

> 【机器人重要规则】OnlyRouter 上同一个模型常有多个「渠道」（名字后缀不同，如 `-de-sp`、`-ab`、`-de-oai`），**价格差很多**。推荐时默认选**该协议下最便宜的可用渠道**。以下价格与协议均已实测验证。

## 核心：先按客户端锁协议，再在该协议里挑最便宜

不同客户端**写死了协议**，只能用对应协议的渠道，填错会报错：
- **Codex** → OpenAI/responses 协议 → 只能用 gpt / deepseek 等 OpenAI 系渠道，**不能填 `-ab` 结尾的**（那是 Anthropic 专用）。
- **Claude Code** → Anthropic/messages 协议 → 用 `claude-*-ab` 或 `claude-*`（Anthropic 渠道），**不能填 gpt/`-de-sp` 这类 OpenAI 渠道**。

> 注：直接调 API 时 OnlyRouter 网关能兼容转换，但 Codex/Claude Code 客户端层面锁死了协议，所以推荐给用户时务必按客户端选对协议的渠道。

## ⚠️ Codex 只支持部分模型（responses 协议，已实测）

Codex 用的是 OpenAI 的 **responses 子协议**，OnlyRouter 上**只有部分模型配置了 responses 协议**，其它填了会报错：
`model "xxx" is not configured for "responses" protocol`。

实测结论：
- ✅ Codex **能用**：`gpt-5.5`、`gpt-5.5-de-sp`、`glm-5.2`、`qwen3.7-max`
- ❌ Codex **不能用**（会报错）：`deepseek-*`、`kimi-*`、`gpt-5.4`、`gpt-5.4-mini`、以及所有 `-ab` 结尾的（Anthropic 专用）

所以**给 Codex 用户推荐模型，只能在上面「能用」的名单里选**，默认推最便宜的 `gpt-5.5-de-sp`。**千万别给 Codex 推 deepseek / kimi**——协议不通，会报错。若用户就想用 deepseek，只能走 Cline/Continue（chat/completions 协议）而不是 Codex。

## Codex 用户：默认推 `gpt-5.5-de-sp`（最便宜）

gpt-5.5 各渠道实测价（输入/输出，$/M tokens）：

| 渠道 | 价格 | 说明 |
|------|------|------|
| **`gpt-5.5-de-sp`** | **$0.4 / $2.4** | ✅ **默认推这个**，最便宜，Codex 可用 |
| `gpt-5.5-ab` | $0.5 / $4 | ⚠️ Anthropic 协议，Codex **用不了**（Claude Code 才行） |
| `gpt-5.5-de-oai` | $3.5 / $21 | 较贵 |
| `gpt-5.5` | $5 / $30 | 官方原价，最贵 |

Codex 配置里 `model = "gpt-5.5-de-sp"` 即可，其余照《官方文档校准-最新配置》。

## Claude Code 用户：默认推 `claude-opus-4-8-ab`（最便宜）

claude-opus-4-8 各渠道实测价：

| 渠道 | 价格 | 说明 |
|------|------|------|
| **`claude-opus-4-8-ab`** | **$0.736 / $3.678** | ✅ **默认推这个**，最便宜，Claude Code 可用 |
| `claude-opus-4-8-de-ccmax` | $1.3 / $6.5 | 较便宜备选 |
| `claude-opus-4-8-de-ant` | $3.75 / $18.75 | 较贵 |
| `claude-opus-4-8` | $4.5 / $22.5 | 官方原价，最贵 |

Claude Code 的 `ANTHROPIC_MODEL` 填 `claude-opus-4-8-ab`。

## 一句话总结（给用户）

- 用 **Codex** → 模型填 **`gpt-5.5-de-sp`**（最便宜）。
- 用 **Claude Code** → 模型填 **`claude-opus-4-8-ab`**（最便宜）。
- 想换别的模型，记住：Codex 别填 `-ab`，Claude Code 只填 `claude-*-ab`/`claude-*`。
- 价格会变，最新以 onlyrouter.ai/models 为准，机器人也会实时拉取。
