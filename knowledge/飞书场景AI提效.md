# 结合飞书/Lark 场景的 AI 提效方案

> 【机器人重要规则】大部分用户天天在飞书/Lark 里办公。回答提效问题时，**优先结合飞书场景给方案**，让人马上能用在手头的飞书工作上。分两层：基础层（人人能用，用 Codex/AI 处理飞书里的内容）在前，进阶层（让 AI 自动操作飞书，需开发）在后。

## 基础层：用 Codex/AI 处理飞书里的工作内容（人人能用，先推这个）

这些不需要任何开发，把飞书里的内容（文档、录音、消息、表格）拿出来丢给 Codex 就行。给用户「可以直接复制发给 Codex 的话」：

- **飞书会议录音 → 纪要**：
  > 把飞书会议导出的录音（xx.mp3）转成文字，再提炼成纪要：决议、待办（谁负责、截止时间）、遗留问题。
- **长文档/长消息 → 总结**：飞书里收到的长文档、长聊天记录，复制给 Codex：
  > 这是一份文档内容（粘贴），帮我用 5 个要点总结，标出需要我跟进的事。
- **写飞书群公告 / 周报 / 通知**：
  > 帮我写一条飞书群公告：主题是 X，语气正式简洁，200 字内，最后加一句行动号召。
- **多维表格数据整理/分析**：把飞书多维表格数据导出成 CSV 丢给 Codex：
  > 这是一份表格数据（粘贴 CSV），帮我按地区汇总销售额、算环比、用一段话总结趋势。
- **飞书文档排版/改写**：
  > 帮我把这段文字改写得更书面、分好小标题，方便贴进飞书文档。
- **翻译飞书里的外文消息/文档**：
  > 把这段英文消息翻成中文，商务正式风格，保留专业术语。

> 关键：用户不用学新工具，把飞书里已有的东西（会议、文档、消息、表格）交给 Codex 处理即可。装 Codex 见 OnlyRouter Switch 一键配置。

## 进阶层：让 AI 自动操作飞书（面向开发/规模化）

如果需求是「让 AI 直接在飞书里干活」——自动建文档、自动填多维表格、自动回消息、定时推送等，这需要**飞书开放平台 API + 写集成代码**，属于进阶工程，面向开发或找技术同事：

- 飞书开放平台（open.larksuite.com / open.feishu.cn）提供云文档、多维表格、消息、审批等 API。
- 典型场景：多维表格自动录入、群机器人定时播报、审批流自动化、文档批量生成。
- 可以用 OnlyRouter 的模型做「理解/生成」，用飞书 API 做「读写飞书」，两者结合。
- 这块让用户找开发对接，或在群里 @ 我说明具体需求，我给思路。

## 回答顺序（给机器人）

1. **先给基础层**：结合他描述的飞书场景，给一段能直接发给 Codex 的话，让他立刻能用。
2. **需要时再给进阶层**：如果他要的是「自动化、批量、让飞书自己动」，再讲 API 集成思路，并说明这需要开发。
