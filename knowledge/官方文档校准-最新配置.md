# OnlyRouter 官方最新配置（已跑通验证，以此为准）

> 来源：onlyrouter.ai/docs（公开页），并经实测验证。若与其他文档冲突，**以本文件为准**。
> 关键变化：官方 Base URL 现在用 **`api.onlyrouter.ai`**（带 api 前缀）。旧的 `onlyrouter.ai/v1` 仍能用（会重定向），但推荐用新地址。

## Base URL（认准这几个）

- 根地址：`https://api.onlyrouter.ai`
- OpenAI / Codex / OpenCode：`https://api.onlyrouter.ai/v1`
- Claude / Gemini 类应用：`https://api.onlyrouter.ai`（不带 /v1）

## 上手四步

1. 注册账号（onlyrouter.ai）
2. 充值 / 找管理员领额度
3. 控制台创建 API Key（`sk-` 开头，只显示一次，别外发）
4. 把 API Key 和 Base URL 填进你的工具

## Codex 配置（config.toml，已验证可用）

编辑 `~/.codex/config.toml`：

```toml
# 基础配置
model_provider = "custom"
model = "gpt-5.5"
model_reasoning_effort = "xhigh"
disable_response_storage = true
personality = "pragmatic"

# 自定义 provider
[model_providers.custom]
name = "onlyrouter"
base_url = "https://api.onlyrouter.ai/v1"
wire_api = "responses"
requires_openai_auth = true
```

改完保存 → **完全退出 Codex** → 用你的 API Key 登录。

> ⚠️ 「Codex」默认指 **Codex App（桌面/CLI）**。用户没特别说明时，按 Codex App 处理。

## OnlyRouter Switch（官方一键配置工具，首选推荐）

> ⚠️ Switch 目前**只支持一键配置 Codex 和 Claude Code 两种客户端**。其它客户端（VS Code 的 Cline/Continue 等）请手动配置。

- 下载地址：https://onlyrouter.ai/download/onlyrouter-switch （Windows / macOS）。
- macOS 若提示「已损坏/无法打开」，终端执行后重开：
  ```
  xattr -cr "/Applications/OnlyRouter Switch.app"
  ```
- 粘贴 API Key → 选模型 → 点「一键配置 Codex / Claude Code」→ 重启客户端验证。

## CC Switch（备选的第三方配置工具）

- macOS Homebrew 安装：
  ```
  brew tap farion1231/ccswitch
  brew install --cask cc-switch
  ```
- 手动填 provider：
  - Claude / Gemini 端点：`https://api.onlyrouter.ai`
  - Codex / OpenCode 端点：`https://api.onlyrouter.ai/v1`
  - API Key：你的 `sk-` key
- 控制台有「填入 CC Switch」按钮，能生成 `ccswitch://` 深链一键导入。

## 直接调 API（OpenAI 格式示例）

```bash
curl https://api.onlyrouter.ai/v1/chat/completions \
  -H "Authorization: Bearer sk-your-key-here" \
  -H "Content-Type: application/json" \
  -d '{"model":"gpt-5.5","messages":[{"role":"user","content":"Hello!"}]}'
```

## 协议支持

- OpenAI Chat Completions 格式
- Anthropic Messages 协议
- 图片 / 视频 / 音频接口（见开发者 API）

## 安全提醒

- 每个客户端单独建一个 Key，别混用。
- API 密钥相当于访问密钥，等于账户里的钱，不要分享、不要发群里。
