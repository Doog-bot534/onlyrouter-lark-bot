把 AI 助手装进你自己的群

给你的企业接一个 飞书 / Lark 答疑机器人：你的客户在你们群里 @ 它， 就能自动获得 OnlyRouter 使用解答。你不用写代码、不用部署服务器，填两个信息就行。

接入前，先在飞书/Lark 后台建个应用（约 5 分钟）
1 创建企业自建应用
打开 open.larksuite.com/app （飞书国内版是 open.feishu.cn/app）→ 点「创建企业自建应用」→ 填个名字和图标。
2 开启「机器人」能力
进应用 → 左侧「添加应用能力」→ 找到「机器人」→ 开启。
3 开通 3 个权限
左侧「权限管理」，搜索并开通：
  • 获取与发送单聊、群组消息（im:message）
  • 以应用身份发消息（im:message:send_as_bot）
  • 读取消息中的图片（im:resource，让机器人能看报错截图）

搜不到英文 code 就按中文名找，认准这三类即可。

4 事件订阅：选「长连接」+ 加接收消息
左侧「事件与回调」→ 订阅方式选 使用长连接接收（很重要，别选发到开发者服务器）→ 添加事件 接收消息 im.message.receive_v1。
5 创建版本并发布 ⚠️
右上角「创建版本」→ 提交发布（企业自建应用一般管理员一键通过）。

⚠️ 这一步最容易漏！不发布版本，前面的权限和事件都不生效，机器人会收不到消息。

6 复制 App ID 和 App Secret
左侧「凭证与基础信息」→ 复制 App ID 和 App Secret 两串字符，粘到下面表单。

App Secret 只用于托管你的机器人，我们加密存储，不会外泄。

填好上面的信息，一键接入 👇
管理已接入的 Bot（改配置 / 暂停 / 删除）

用接入成功时给你的「管理令牌」来操作。

常见问题
接入后，我的客户怎么用？
把机器人拉进你们的群（群设置 → 群机器人 → 添加机器人 → 选你的应用），客户在群里 @ 它提问即可。
群里 @ 了没反应？
八成是没「创建版本并发布」（第 5 步），或事件订阅没选长连接。回后台检查并重新发布。
App Secret 交给你们安全吗？
只用于托管你的机器人，我们用 AES 加密存储，日志也不会打印。你随时可以在飞书后台重置 Secret 使旧的失效。
会产生费用吗？
机器人回答用的是 OnlyRouter 的模型额度。托管本身不额外收费。
bug 反馈是怎么回事？
机器人若判断用户遇到的是 OnlyRouter 平台 bug，会静默同步给我们帮助改进产品，不会打扰你的群。

接入即表示同意由 OnlyRouter 托管运行你的机器人。凭证加密存储，仅用于此用途。
